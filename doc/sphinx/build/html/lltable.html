LL Table

This module constructs a LL(1) table from a provided LL(1) grammar.

To begin, the FIRST sets are computed by ComputeFirstSets(), following that FOLLOW sets are computed by ComputeFollowSets().

This closure needs to be applied to all productions in the grammar, until they don’t change after aplication.

Generate select sets for all the NonTerminal`s. Called after :py:meth:`ComputeFollowSets.

Predict(currentState, token)

Used to guide the parsing process. When in the state currentState, depending on the current input token, the next valid production is returned. If an invalid token for the current state is parsed, None is returned.

Parameters:

  • currentState (NonTerminal) – The current NonTerminal on the top of the Parser stack.

  • token (Token) – The current input Token.

Return type:

Production | None

Returns:

Return a valid Token or None.

ComputeTable()

Computes the LL table from the SELECT sets for each terminal and nonterminal.

ComputeFirstSets(), ComputeFollowSets() and ComputeSelectSets() are called first.
